# Terraform-DevOps-Study
DevOps 엔지니어를 위한 Terraform 및 IaC 도구 학습 리포지토리입니다.

### 1. 스터디 운영 규칙
- **기간 및 일정**: 총 3주 진행 (일정 협의 필요), 프로젝트 적용 목표로 합니다.
- **진행 방식**:
  - 매주 정해진 섹션의 강의를 수강하고 학습 내용을 정리합니다.
  - 각 스터디원은 담당한 파트에 대해 발표 자료(마크다운)를 준비합니다.(옵시디언 사용)
- **과제 제출**:
  - 매주 정기 모임 전날 자정까지 PR을 통해 제출합니다.
  - 정리 툴은 자유이나, 깃허브 PR을 위해 마크다운(Obsidian 등)을 권장합니다.
- **강의**: [Udemy: Mastering Terraform & Docker & Packer on AWS](https://www.udemy.com/course/best-devops-terraform/?couponCode=CP251120G2V2)

### 2. 세션 진행 (20~30분 내외)
담당자는 아래 목차를 참고하여 세션을 준비해주세요.
- **개요 및 핵심 개념**: 해당 섹션에서 다루는 기술의 정의와 필요성
- **코드 및 문법 설명**: HCL 문법, 주요 리소스 옵션 설명
- **실습 및 데모**: 실제 코드로 인프라가 배포되는 과정 시연
- **Exam Tip**: 자격증 시험(Terraform Associate)에서 주의할 점
- **Q&A**: 학습하며 생겼던 의문점 공유 및 토론

### 3. 레포지토리 컨벤션
- **섹션 선정**: Issue 탭에 `[주차] 섹션 이름`으로 등록하여 선점합니다.
  - 예: `[1주차] 테라폼 기초`
- **브랜치 전략**: `이름/챕터`
  - 예: `gusionling/chapter1`, `swthewhite/chapter2`
- **커밋 컨벤션**: `유형: 작업 사항`
  - 유형: `add`, `update`, `delete`, `docs`
  - 예: `add: 테라폼 기초 문법 정리`
